## Field-by-Field Analysis

### Always Strip (14 fields)

These fields are either redundant with the query/metadata, always null, or purely technical formatting hints with no value to the LLM.

| Field | Observed Values | Reason to Strip |
|---|---|---|
| `DATABASE_ID` | Always matches query param | Redundant -- already in metadata |
| `INDICATOR` | Always matches query param | Redundant -- already in metadata |
| `DATA_SOURCE` | Matches `DATABASE_ID` or `null` | Redundant |
| `TIME_FORMAT` | `"P1Y"`, `"602"`, `"610"`, `null` | ISO duration code, no LLM value |
| `UNIT_MULT` | Always `0` | Multiplier, never non-zero in survey |
| `COMMENT_OBS` | Always `null` | No data observed |
| `OBS_STATUS` | Always `"A"` (actual) | Uniform, no signal |
| `OBS_CONF` | Always `"PU"` (public) | Uniform, no signal |
| `DECIMALS` | `"1"`, `"2"`, `null` | Formatting hint |
| `LATEST_DATA` | `true`/`false` | Redundant with `TIME_PERIOD` sorting |
| `FREQ` | `"A"`, `"M"`, `"_O"` | Already conveyed by metadata periodicity |
| `UNIT_TYPE` | Always `null` | No data observed |
| `COMP_BREAKDOWN_3` | Always `"_Z"` in all 16 databases | Never carries data |
| `COMMENT_TS` | Indicator name or `null` | **Promoted to metadata** (see below) |

### Always Keep (5 fields)

| Field | Purpose |
|---|---|
| `OBS_VALUE` | The actual data value |
| `TIME_PERIOD` | Time dimension (year or date) |
| `REF_AREA` | Country/region code |
| `UNIT_MEASURE` | Unit context (e.g. `"PT"`, `"USD_K_2015"`, `"PS"`) |
| `claim_id` | Verification hash (generated by MCP, not from API) |

### Conditionally Keep (5 fields)

These carry real data in specific databases but are `_T` (total) or `_Z` (not applicable) in most. Keep only when value is meaningful.

| Field | Non-Trivial In | Example Values | Rule |
|---|---|---|---|
| `SEX` | WB_HCP, WB_SSGD, WB_GS | `"F"`, `"M"` | Keep if not `_T` and not `_Z` |
| `AGE` | WB_SSGD, OECD_IDD | `"Y15T24"`, `"Y18T65"`, `"Y_GE25"`, `"Y_GE60"`, `"Y_GT65"`, `"Y30T59"`, `"Y15T29"` | Keep if not `_T` and not `_Z` |
| `URBANISATION` | WB_SSGD | `"RUR"`, `"URB"` | Keep if not `_T` and not `_Z` |
| `COMP_BREAKDOWN_1` | IPC_IPC, OECD_BROADBAND, WB_SE4ALL, WEF_TTDI | `"IPC_IPC_CURRENT"`, `"OECD_BROADBAND_INDICATOR_TYPE_FBB"`, `"WB_SE4ALL_TYPE_RENEWBLS"`, `"WEF_TTDI_VAL"` | Keep if not `_T` and not `_Z` |
| `COMP_BREAKDOWN_2` | IPC_IPC, OECD_IDD | `"IPC_IPC_PHASE1"`...`"PHASE5"`, `"OECD_IDD_DEFINITION_D_CUR"` | Keep if not `_T` and not `_Z` |

### Edge Case: `AGG_METHOD`

| Observed Values | Databases |
|---|---|
| `"_Z"` | WB_WDI, WB_SSGD, most others |
| `null` | WB_GS, WB_HCP, IPC_IPC, OECD_BROADBAND |
| `"W_AVG"` | WB_SE4ALL |

**Decision**: Strip. `W_AVG` appeared in only 1 database and is conveyed by the indicator's metadata definition. Not worth the per-row overhead.

### Edge Case: `COMMENT_TS`

Contains the indicator name (e.g., "GDP per capita (constant 2015 US$)"). Repeats identically on every row when present, `null` in many databases.

**Decision**: Promote first non-null value to `metadata["indicator_description"]`, do not include per row.

### Edge Case: `IMF_FAS` Missing `SEX` Dimension

`IMF_FAS` does not expose a `SEX` field in its disaggregation response. However, the raw data rows still contain `SEX` with some value. The conditional logic handles this -- if the value is `_T` or `_Z`, it's stripped regardless.

## Disaggregation Dimension Summary

Full map of all non-trivial disaggregation values observed:

### SEX
```
_T  = Total (all databases, default)
F   = Female (WB_HCP, WB_SSGD, WB_GS)
M   = Male (WB_SSGD, WB_GS)
```

### AGE
```
_T      = Total (default)
Y15T24  = Ages 15-24 (WB_SSGD)
Y15T29  = Ages 15-29 (WB_SSGD)
Y18T65  = Ages 18-65 (OECD_IDD)
Y30T59  = Ages 30-59 (WB_SSGD)
Y_GE25  = Ages 25+ (WB_SSGD)
Y_GE60  = Ages 60+ (WB_SSGD)
Y_GT65  = Ages 65+ (OECD_IDD)
```

### URBANISATION
```
_T  = Total (default)
RUR = Rural (WB_SSGD)
URB = Urban (WB_SSGD)
```

### COMP_BREAKDOWN_1
```
_T  = Total (WB_SSGD)
_Z  = Not applicable (WB_WDI, WB_GS, WB_HCP, WB_HNP, WB_ESG, ITU_DH, WJP_ROL)
IPC_IPC_CURRENT                    (IPC_IPC)
OECD_BROADBAND_INDICATOR_TYPE_FBB  (OECD_BROADBAND)
WB_SE4ALL_TYPE_RENEWBLS            (WB_SE4ALL)
WB_SE4ALL_TYPE_MDRNRENWBLS         (WB_SE4ALL)
WEF_TTDI_VAL                       (WEF_TTDI)
WEF_TTDI_RNK                       (WEF_TTDI)
WEF_TTDI_SCR                       (WEF_TTDI)
```

### COMP_BREAKDOWN_2
```
_T  = Total (WB_SSGD)
_Z  = Not applicable (most databases)
IPC_IPC_PHASE1 through PHASE5      (IPC_IPC)
OECD_IDD_DEFINITION_D_PREV         (OECD_IDD)
OECD_IDD_DEFINITION_D_CUR          (OECD_IDD)
OECD_IDD_DEFINITION_D_INC          (OECD_IDD)
```

### COMP_BREAKDOWN_3
```
_Z  = Not applicable (ALL databases -- always trivial)
```

## Design Decisions

1. **Allowlist approach, not blocklist.** Only explicitly listed fields are kept. Unknown fields from future API changes are stripped by default. This is safer for token savings.

2. **Trivial values = `_T` and `_Z` only.** These are the SDMX standard codes for "total" and "not applicable". Any other value is assumed meaningful and retained.

3. **`claim_id` survives stripping.** The hash is computed on the raw row before stripping, so verification integrity is preserved.

4. **`COMMENT_TS` promoted, not dropped.** Its value is useful context (indicator description) but should appear once in metadata, not repeated per row.

5. **`COMP_BREAKDOWN_3` always stripped.** Confirmed `_Z` across all 16 databases. No data source in the system uses this dimension.

6. **`AGG_METHOD` stripped despite `W_AVG` in SE4ALL.** One database having a non-trivial value does not justify per-row overhead. The aggregation method is metadata, not row-level data.
